Question 1

public class Main {
    public static void main(String[] args) {
        int String = 10;
        System.out.print(String);
    }
}

Topic: Variable Naming

What will be the outcome when you try to compile this program?

  • A. Compilation Error: 'String' is a reserved keyword
  • B. Compilation Error: 'int' cannot be used as an identifier
  • C. 10
  • D. Exception

Answer: C. 10

Explanation: The code will compile and run successfully because although `String` is a reserved keyword, in this context it is used as a variable name. The variable `String` is assigned the value 10, which will be printed by the `System.out.print` statement.

Question 2

public class Main {
    public static void main(String[] args) {
        byte a = "durga";
        System.out.print(a);
    }
}

Topic: Data Type

What will be the outcome when you try to compile this program?

  • A. Compilation Error: incompatible types: String cannot be converted to byte
  • B. Compilation Error: incompatible types: possible lossy conversion
  • C. 0
  • D. Exception

Answer: A. Compilation Error: incompatible types: String cannot be converted to byte

Explanation: The code will produce a compilation error because a String cannot be assigned to a byte variable.

Question 3

public class Main {
    public static void main(String[] args) {
        byte a = 10.5;
        System.out.print(a);
    }
}

Topic: Data Types

What will be the outcome when you try to compile this program?

  • A. Compilation Error: incompatible types: possible lossy conversion from double to byte
  • B. Compilation Error: incompatible types: possible lossy conversion from float to byte
  • C. 10
  • D. Exception

Answer: A. Compilation Error: incompatible types: possible lossy conversion from double to byte

Explanation: The code will produce a compilation error because 10.5 is a double literal, which cannot be directly assigned to a byte variable without explicit casting.

Question 4

public class Main {
    public static void main(String[] args) {
        byte a = 128;
        System.out.print(a);
    }
}

Topic: Data Types

What will be the outcome when you try to compile this program?

  • A. Compilation Error: constant expression required
  • B. Compilation Error: possible lossy conversion from int to byte
  • C. 128
  • D. Exception

Answer: B. Compilation Error: possible lossy conversion from int to byte

Explanation: The code will produce a compilation error because 128 is outside the valid range for a byte (-128 to 127). Assigning 128 to a byte variable requires explicit casting to handle the value properly.
