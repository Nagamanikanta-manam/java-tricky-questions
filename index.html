Question 1

public class Main {
    public static void main(String[] args) {
        int String = 10;
        System.out.print(String);
    }
}

Topic: Variable Naming

What will be the outcome when you try to compile this program?

  • A. Compilation Error: 'String' is a reserved keyword
  • B. Compilation Error: 'int' cannot be used as an identifier

  • D. Exception

Answer: C. 10

Explanation: The code will compile and run successfully because although `String` is a reserved keyword, in this context it is used as a variable name. The variable `String` is assigned the value 10, which will be printed by the `System.out.print` statement.

  • B. 10..8..16

Thus, the output will be `10..8..16`.

Question 17

public class Main {
    public static void main(String[] args) {
        int a = 1_2_2_3;
        
        System.out.println(a);
    }
}

Topic: Underscores in Numeric Literals

What will be the output of this program when it is run?

  • A. 1223
  • B. Compilation Error: Invalid underscore placement
  • C. 1_2_2_3
  • D. Exception

Answer: A. 1223

Explanation: In Java, underscores (`_`) can be used in numeric literals to improve readability. They do not affect the actual value of the number. The value of `a = 1_2_2_3` is interpreted as `1223` and will be printed as such. Java allows underscores in numeric literals as long as they are placed between digits, but not at the start or end of the literal.
